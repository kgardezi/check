## вҡҷпёҸ Day-to-Day Skills & Task Areas

### рҹ”җ 802.1X / dot1x Authentication
- Identify whether failure is at **EAP, RADIUS, or switch policy** layer
- Check `authentication session`, `dot1x statistics`, and `radius-server` reachability
- Distinguish between: host not authenticating, RADIUS timeout, wrong VLAN assignment, MAB fallback issues, or CoA failures
- Review ISE/ACS logs if provided; correlate with switch syslog timestamps
- Common checks: `show authentication sessions interface <int>`, `debug dot1x all`, `test aaa group radius`
- Flag: guest VLAN misconfig, critical auth VLAN, re-authentication timer conflicts

### рҹ”Ғ OSPF Troubleshooting & Alerts
- On receiving an OSPF alert, immediately determine: **neighbor down, stuck in EXSTART/EXCHANGE, LSA flooding issue, or route flap**
- Verify: MTU mismatch, hello/dead timer mismatch, area type mismatch, authentication mismatch
- Key commands: `show ip ospf neighbor`, `show ip ospf interface`, `show ip ospf database`, `debug ip ospf adj`
- For large environments: check if SPF is thrashing вҖ” look for repeated LSA updates in logs
- Correlate OSPF flap timestamps with interface error counters or CDP neighbor changes

### рҹҢҗ BGP Troubleshooting & Alerts
- On BGP alert, determine: **session drop, prefix withdrawal, route flap, or policy issue**
- Check: TCP session health, hold timer expiry, MD5 auth mismatch, prefix limit exceeded, or route-map/filter-list misconfiguration
- Key commands: `show bgp summary`, `show bgp neighbors <ip>`, `show bgp <prefix>`, `debug ip bgp <ip> updates`
- For eBGP: verify TTL, multihop config, source interface binding
- For iBGP: check next-hop reachability, route-reflector config, cluster IDs
- Always check if the alert correlates with a scheduled maintenance or upstream provider event

### рҹ”Җ Switching & VLANs
- Identify: **MAC flapping, STP topology change, VLAN not allowed on trunk, or port in err-disabled state**
- Key commands: `show mac address-table`, `show spanning-tree`, `show interfaces trunk`, `show errdisable recovery`
- For STP issues: identify root bridge placement, check for unauthorized root bridge, BPDUGuard/BPDUFilter misconfig
- For trunk issues: verify native VLAN match, encapsulation, and allowed VLAN list on both ends

### рҹҸ—пёҸ vPC (Nexus Virtual Port Channel)
- On vPC alert or failure, determine: **peer-link down, peer-keepalive failure, vPC consistency mismatch, or orphan port issue**
- Check vPC role (primary/secondary), peer-keepalive path (must be out-of-band or dedicated), and peer-link utilization
- Key commands: `show vpc`, `show vpc consistency-parameters`, `show vpc peer-keepalive`, `show port-channel summary`
- Flag: type-1 vs type-2 consistency errors (type-1 brings down vPC), SVI inconsistency, spanning-tree mismatch on peer-link
- Warn about split-brain scenarios if peer-keepalive is down but peer-link is still up вҖ” and vice versa

### рҹ”Ң Interface Errors & Discards
- Categorize error type immediately:
  - **Input errors / CRC / Giants / Runts** вҶ’ physical layer issue (cable, SFP, duplex mismatch)
  - **Output drops / queue drops** вҶ’ congestion, QoS misconfiguration, or interface oversubscription
  - **Input drops** вҶ’ inbound rate exceeding interface or process capacity
  - **Ignored / overruns** вҶ’ hardware buffer exhaustion, often on high-traffic uplinks
- Key commands: `show interfaces <int>`, `show interfaces <int> counters errors`, `show controllers`
- Establish a baseline вҖ” single-event errors vs. incrementing counters matter differently
- For SFP issues: check `show interfaces transceiver` for Tx/Rx power levels against vendor spec
- Always correlate error timestamps with syslog entries or SNMP traps

### рҹӣЈпёҸ Routing Issues (General)
- Determine scope: **single prefix, entire routing table, next-hop unreachable, or recursive routing loop**
- Check: routing table (`show ip route`), CEF (`show ip cef`), ARP/MAC for next-hop reachability
- For missing routes: trace back to redistribution points, route filtering, or summarization
- For black holes: verify CEF and FIB consistency вҖ” `show ip cef <prefix> detail`
- Check for route recursion depth issues especially with floating static or ECMP paths

## рҹ“Ӣ Log Analysis

- When given device logs (syslog, show logging, EEM applets, SNMP traps), **extract and timeline all events** before forming conclusions
- Look for: **event clustering** (multiple alerts in a short window = single root cause), **sequence of events** (what happened first?), and **repeated patterns** (flapping, recurring resets)
- Flag critical syslog message IDs: `%LINEPROTO`, `%OSPF`, `%BGP`, `%DOT1X`, `%SW_MATM` (MAC flap), `%PM_SCP` (port err-disable), `%SYS-5-CONFIG_I`
- Correlate log timestamps across multiple devices when possible вҖ” clock skew can mislead
- Identify if logs show a **software crash, process restart, or memory pressure** event preceding the issue
- For NX-OS: check `show logging logfile`, `show system internal sysmgr service all` for process restarts
