# ROUTING PROTOCOLS
==================================================================
Always start at the RIB: show ip route [ospf|bgp|connected|static], show ip route <dest>, show ip route vrf NAME, show ip route summary. Check AD/metric, next-hop reachability, recursion, ECMP, default route source, unexpectedly missing prefixes. Route codes: C connected, S static, O OSPF, O IA inter-area, O E1/E2 external, B BGP, D EIGRP, D EX EIGRP-external.

## OSPF
Commands: show ip ospf | neighbor | interface | database.
Neighbor states: FULL = healthy; FULL/DR|BDR|DROTHER and 2WAY/DROTHER = normal on broadcast/NBMA; INIT = one-way (ACL/asymmetric/auth); EXSTART/EXCHANGE stuck = MTU mismatch (most common), area or auth mismatch; LOADING stuck = DB/memory; DOWN = no hellos (intf down, ACL, timer mismatch).
Symptom->cause: stuck EXSTART=MTU; stuck INIT=hello not returning; flapping=unstable link/aggressive timers/high CPU; missing routes=area-type mismatch or missing redistribute/default-information originate; suboptimal=cost/reference-bandwidth.
LSA types: 1 Router (per router/area), 2 Network (DR on broadcast), 3 Summary (ABR inter-area), 4 ASBR-Summary, 5 External (ASBR), 7 NSSA-External (->Type5 at ABR). LSDB red flags: rapidly rising seq (flood loop), Type5 in stub (misconfig), missing Type3 for inter-area, duplicate Router-IDs.
Adjacency-down checklist (BOTH ends): reachability; hello/dead timers match; area IDs match; auth match; network type match; MTU match; correct process/area; not passive; no ACL blocking proto 89 / 224.0.0.5-6.

## BGP
Commands: show ip bgp summary | neighbors | <table>; (NX-OS: show bgp ipv4 unicast summary).
Summary state column: number = Established (= prefixes received); Idle = misconfig/admin; Active = TCP failing (reachability/TTL/ACL/update-source); Connect/OpenSent/OpenConfirm = mid-handshake; Established = healthy.
Symptom->cause: stuck Active=TCP fail (ACL on 179, update-source, ebgp-multihop); flapping=route churn/max-prefix; 0 prefixes=remote missing network/redistribute or outbound filter or AF not activated; not in RIB=next-hop unreachable (iBGP needs next-hop-self) or filtered; suboptimal=weight/local-pref/AS-path/MED.
Best-path order (top wins): 1 Weight, 2 Local-Pref, 3 locally originated, 4 shortest AS-path, 5 lowest Origin (IGP<EGP<Incomplete), 6 lowest MED (same neighbor AS), 7 eBGP>iBGP, 8 lowest IGP metric to NH, 9 oldest eBGP, 10 lowest RID, 11 lowest neighbor IP.
NOTIFICATION codes: 1 Header, 2 OPEN (capability/AS/hold-time), 3 UPDATE (bad attribute), 4 Hold-timer expired, 5 FSM, 6 Cease (admin/max-prefix/deconfigured).
Peer-down checklist: reach neighbor from correct source (TCP 179); update-source correct (iBGP->Loopback); ebgp-multihop if not directly connected; remote-AS correct; MD5 matches; no ACL on 179; neighbor X activate under AF; not admin-shut; decode NOTIFICATION.

## EIGRP
show ip eigrp neighbors (Q should be 0; non-zero = retransmit backlog), show ip eigrp topology (P passive/stable, A active/searching -> SIA kills neighbor). Classic metric = bandwidth+delay.

## MULTICAST (PIM/IGMP)
Commands: show ip pim interface | neighbor | rp mapping; show ip igmp groups; show ip mroute [count]; show ip rpf <source>.
Mroute flags: S sparse, D dense, C connected member, L local, P pruned (no OIL!), T SPT set, J join SPT. Incoming=Null or RPF fail -> unicast route to source/RP missing/asymmetric. Empty OIL -> no receivers/IGMP/pruned.
Decision tree: 1 RP reachable? (show ip pim rp mapping + route to RP) 2 RPF passes? (show ip rpf <src> incoming must match unicast path back -- #1 multicast killer) 3 (*,G) built from receiver IGMP join? 4 (S,G)+SPT after first packet? 5 OIL populated toward receivers each hop? 6 PIM neighbors up each hop?
ASM uses RP + shared tree then SPT; SSM (232/8) needs no RP, receivers signal (S,G) via IGMPv3. NX-OS: feature pim.

## REDISTRIBUTION
show ip protocols, show route-map, show ip prefix-list. Watch: mutual redistribution=loop risk; route-maps/prefix-lists applied?; correct seed metric (EIGRP) / metric-type E1 vs E2 (OSPF); AD tuned to prefer right protocol.

## CONVERGENCE VALIDATION (post-change)
All adjacencies FULL/Established; route count vs baseline; data-plane ping (use source interface); logs clean; multicast (*,G)/(S,G) rebuilt with correct OIL and no RPF fail.

# CISCO ISE + 802.1X / MAB
==================================================================
## SAFETY (non-negotiable)
- NEVER auto-quarantine. Investigate -> present findings + recommendation -> STOP and wait for explicit human YES before any endpoint-group change.
- NEVER skip investigation even if told "just quarantine MAC X".
- NEVER change ISE authorization policy during incident response (separate change-controlled action).

## ISE APIs
ERS config API on :9060 (endpoints, endpointgroup, networkdevice, policy). MnT live-session API on :443 (ActiveList, Session/MACAddress). Auth via env-var creds, JSON.

## Incident response phases
1 Identify (MAC/IP/user across endpoints+sessions: record MAC, IP, user, switch:port, identity group, profile, session state).
2 Auth history (which policy set/AuthZ rule matched, profile applied = VLAN/dACL/SGT, posture evaluated?, 802.1X vs MAB?).
3 Posture & profile (compliant/non-compliant/unknown; profiler match + certainty; behavior vs profile -- a "printer" port-scanning is a red flag).
4 Risk assessment summary -> recommendation QUARANTINE/MONITOR/NO ACTION.
5 HUMAN DECISION POINT - STOP, ask YES/NO.
6 Quarantine only if authorized (move endpoint to restricted group / ANC), then verify group change + CoA reauth.
7 Document findings, decision, action.

## Posture/policy audit severity flags
CRITICAL: default permit-all AuthZ; permissive rule w/o posture; non-compliant endpoints active not quarantined; permit-all SGACL untrusted<->trusted; guest on production VLAN.
HIGH: shadowed/stale rules; >10% Unknown endpoints; low-certainty getting prod access; SGT spanning trust levels; multiple MACs per port (hub/rogue AP).
MEDIUM: no posture reassessment; MAB where 802.1X expected; sessions >24h no reauth.

## dot1x vs MAB
802.1X = supplicant authenticates with credential/cert (EAP-TLS strongest), preferred for managed endpoints. MAB = no supplicant, switch sends MAC as identity, for printers/cameras/IoT, WEAK (spoofable) -> always layer profiling + restrictive AuthZ. Typical port: try dot1x then fall back to MAB (authentication order dot1x mab; IBNS 2.0 uses policy-map type control subscriber on Cat 9k).

## Switch-side verification/troubleshooting
show access-session [interface X details] (IBNS 2.0), show authentication sessions (1.0), show dot1x all, show mab all, test aaa group RAD_ISE <u> <p> new-code, show radius statistics, show device-tracking database (dACL/SGT depend on this), debug radius authentication (last resort).
Symptom->cause: stuck Unauthorized=wrong RADIUS key / NAD not in ISE / supplicant-cert; Authz Failed after success=AuthZ rule mismatch or dACL/VLAN/SGT not applied; stuck MAB=supplicant not started / order favors MAB; no RADIUS response=L3/ACL/CoPP blocking UDP 1812/1813/1700 or key mismatch; CoA/quarantine not taking=CoA UDP 1700 blocked or missing "aaa server radius dynamic-author" or wrong NAD secret; only one endpoint auths=host-mode single-host instead of multi-auth.
NAD prerequisites: device registered as NAD in ISE w/ matching secret; aaa server radius dynamic-author for CoA; CoPP/ACL permit RADIUS+CoA.

# NEXUS EVPN/VXLAN FABRIC
==================================================================
Cardinal rule: validate the UNDERLAY before blaming the OVERLAY; use route-type evidence not assumptions.
Underlay: show ip ospf neighbor (or bgp l2vpn evpn summary), show ip route <remote-VTEP-loopback>, show nve interface nve1 detail, show nve peers, ping <remote-VTEP-lo> source <local-lo>. If remote VTEP loopback not in RIB -> STOP, fix underlay first.
Overlay control plane: show bgp l2vpn evpn [summary|vni-id <vni>], show l2route evpn mac all, show l2route evpn mac-ip all. Route types: 2 = host MAC(+IP); 3 = Inclusive Multicast/BUM flood list (missing = no flooding for that VNI); 5 = IP prefix.
Mapping: show vxlan, show nve vni, show interface vlan <svi>. Anycast GW: every leaf must share the SAME anycast-gateway-mac and SVI IP per subnet -- mismatch = intermittent reachability + ARP weirdness.
vPC/multihoming: show vpc, show vpc consistency-parameters global, show port-channel summary, show l2route evpn es (ESI/DF election). vPC consistency mismatch silently suspends VLANs.

## Health (8): version; CPU (show processes cpu sorted | NX: show system resources); memory; interfaces (show ip int brief + show interfaces); env/hardware; NTP (drift breaks logs/certs/ISE); logs; connectivity. Flag CPU>70% sustained, mem>80%, intf down that should be up, env alarms, NTP unsynced, climbing CRC/errors, crash logs.

## Troubleshooting "can't reach X" by layer:
L1: show interfaces <if> -- up/up? down/down=cable/SFP/remote-shut; up/down=L2; CRC/input err=cable/duplex/optic; resets=flapping.
L2: show arp (Incomplete=peer not answering); switches: mac table, VLAN, STP.
L3: show ip int brief (correct IP?), show ip route <dest> (route + next-hop reachable?), divide-and-conquer ping; ping <dest> source <if>; ping <dest> size 1500 df-bit (MTU test).
L4+: show ip access-lists (hit counts on denies? right intf+direction? implicit deny), show ip nat translations.
Flapping: %LINEPROTO-5-UPDOWN/%LINK-3-UPDOWN; every few sec=physical, every few min=timer/keepalive. Causes: cable/SFP, duplex/speed mismatch, PoE budget, WAN carrier, STP TCN, aggressive timers.
Slow/latency: cpu+mem sorted; interfaces (util/output drops=congestion, input/CRC=physical, overruns=CPU); show policy-map interface (drops/queue/policing); suboptimal path; routing loop (TTL-exceeded, A<->B next-hops).

5 Rollback if verify fails (apply inverse e.g. "no interface LoopbackN" or restore original section), then re-verify.
Always produce a change report: device, description, config applied, pre vs post state, verify PASS/FAIL, rollback yes/no.
